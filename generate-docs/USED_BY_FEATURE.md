# "Used By" Feature Documentation

## Overview

The "Used By" feature adds reverse reference sections to Office.js API documentation, showing where types are used throughout the API. This helps developers discover related APIs and understand type relationships.

## Implementation

### Location
- **File**: `generate-docs/scripts/yaml-office-processor.ts`
- **Lines Added**: ~350 lines

### How It Works

1. **Index Building Phase**
   - Reads all `*.api.json` files produced by API Extractor
   - Analyzes excerpt tokens to find type references
   - Builds a reverse index: `type UID → list of members that use it`
   - Tracks property types, method return types, and parameter types

2. **YAML Enhancement Phase**
   - For each YAML file, looks up the type's UID in the index
   - If references exist, generates a "Used By" markdown section
   - Injects the section into the `remarks` field
   - Positions it before "Examples" section if present

### Features

- **Cross-Package References**: Groups references by package (Office, Excel, Outlook, etc.)
- **Deduplication**: Removes duplicate method overload entries
- **Smart Filtering**: Excludes built-in types (Promise, Array)
- **Context Information**: Shows whether type is used as property/return/parameter
- **Versioning Support**: Works with all versioned API folders

## Example Output

### Simple Reference (Office.ContextInformation)

```yaml
remarks: |-
  \*\*Important\*\*: In Outlook, this object is available starting with...

  #### Used by

  - <xref uid="office!Office.Context#diagnostics:member" /> (property type)
```

### Multiple References (Excel.AggregationFunction)

```yaml

  - <xref uid="excel!Excel.Interfaces.DataPivotHierarchyUpdateData#summarizeBy:member" /> (property type)
```

## Architecture

### Data Flow

```
JSON Files (API Extractor)
  ↓
buildUsedByIndex()
  ↓
UsedByIndex (type UID → UsedByReference[])
  ↓
processYamlFile()
  ↓
injectUsedBySection()
  ↓
Enhanced YAML Files
```

### Key Functions

- `buildUsedByIndex()` - Builds the reverse reference index from JSON files
- `analyzeMembers()` - Recursively traverses API structure
- `analyzeMemberTypeReferences()` - Extracts type references from properties/methods
- `analyzeExcerptTokens()` - Parses excerpt tokens for type references
- `injectUsedBySection()` - Adds "Used By" markdown to YAML items
- `generateUsedBySection()` - Formats the markdown output
- `deduplicateMethodOverloads()` - Removes duplicate overload entries
- `groupByPackage()` - Groups references by package for display

## Edge Cases Handled

1. **Method Overloads**: Deduplicates `member(1)`, `member(2)` → `member`
2. **Built-in Types**: Skips `Promise`, `Array`, etc. (UIDs starting with `!`)
3. **Cross-Package**: Groups and labels references from different packages
4. **Versioned APIs**: Works correctly with `excel_1_5`, `outlook_1_3`, etc.
5. **Empty Results**: No section added if type has no references
6. **Existing Content**: Correctly positions before Examples, after remarks

## Performance

- **Build Time**: Adds ~2-3 seconds to total YAML processing
- **Index Memory**: ~1-2 MB for full index
- **File I/O**: Single pass through JSON files, single pass through YAML files

## Maintenance

The feature is self-contained in `yaml-office-processor.ts` with no external dependencies. Updates to the API structure are automatically reflected when the JSON files are regenerated by API Extractor.
